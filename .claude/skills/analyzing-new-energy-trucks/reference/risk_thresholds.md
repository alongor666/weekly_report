# 风险阈值标准

## 机构风险分级

### 赔付率分级

| 等级 | 赔付率范围 | 标识 | 行动策略 |
|------|-----------|------|---------|
| 优秀 | < 50% | 🟢 | 维持策略，适度拓展 |
| 良好 | 50-60% | 🟢 | 正常经营，持续监控 |
| 中等 | 60-70% | 🟡 | 关注趋势，控制增长 |
| 预警 | 70-80% | 🟠 | 加强审核，限制规模 |
| 危险 | 80-100% | 🔴 | 立即整改，暂停新单 |
| 极危 | > 100% | ⚫ | 紧急干预，全面停保 |

### 边际贡献率分级

| 等级 | 边际贡献率 | 标识 | 盈利状态 |
|------|-----------|------|---------|
| 优秀 | > 12% | 🟢 | 健康盈利 |
| 良好 | 8-12% | 🟡 | 盈利可接受 |
| 中等 | 6-8% | 🟠 | 微利警戒 |
| 预警 | 4-6% | 🔴 | 濒临亏损 |
| 危险 | 0-4% | ⚫ | 几乎不盈利 |
| 亏损 | < 0% | ☠️ | 亏损状态 |

## 异常检测阈值

### 1. 单周暴涨检测

**触发条件**:
```python
if current_loss_ratio > prev_loss_ratio * 1.15:
    alert_level = "medium"

if current_loss_ratio > prev_loss_ratio * 1.30:
    alert_level = "high"

if current_loss_ratio > prev_loss_ratio * 1.50:
    alert_level = "critical"
```

**行动建议**:
- 中度: 调查当周大额案件
- 高度: 立即复核定损流程
- 极危: 暂停该机构承保

### 2. 连续恶化检测

**触发条件**:
```python
# 连续3周赔付率上升
if all(loss_ratios[i] < loss_ratios[i+1] for i in range(-4, -1)):
    alert("连续3周恶化")
```

**行动建议**:
- 召开风险评估会议
- 调查业务质量问题
- 收紧核保政策

### 3. 案均赔款异常

**触发条件**:
```python
period_avg = np.mean(avg_claims)

if current_avg_claim > period_avg * 1.5:
    alert("案均赔款异常")
```

**关注重点**:
- 是否有大额案件集中
- 定损标准是否合理
- 欺诈风险排查

## 新能源特殊阈值

### 电池风险指标

**大额案件占比**:
```python
large_claim_ratio = (
    claims[claims > 50000].sum() /
    total_claims * 100
)

# 评估标准
if large_claim_ratio > 40%:
    risk_level = "high"  # 电池风险过高
```

**评估标准**:
- 正常: < 20%
- 关注: 20-30%
- 预警: 30-40%
- 危险: > 40%

### 出险率特殊阈值

新能源货车出险率普遍偏高，调整标准：

| 25% | 50% | 良好 |
| 50% | 75% | 正常 |
| 75% | 100% | 偏高 |
| 100% | 125% | 危险 |

| 城市等级 | 充电桩密度 | 风险系数 | 赔付率调整 |
|---------|-----------|---------|-----------|

**应用示例**:
```python
# 调整后的赔付率评估
adjusted_loss_ratio = actual_loss_ratio / infrastructure_factor

# 例如：三线城市实际赔付率120%
# 调整后：120% / 1.5 = 80%（仍处于危险区间）
```

## 趋势预警阈值

### 波动性指标

**赔付率标准差**:
```python
volatility_std = np.std(weekly_loss_ratios)

# 分级
if volatility_std < 10:
    stability = "稳定"
elif volatility_std < 20:
    stability = "中等波动"
else:
    stability = "高波动，需关注"
```

### 增长率预警

**保费规模增长**:
- 健康增长: 5-15%
- 快速增长: 15-30% (关注质量)
- 激进增长: >30% (高风险)

**赔款增长**:
- 如赔款增长 > 保费增长 × 1.2
- 则触发质量恶化预警

## 阈值更新机制

### 季度评估

每季度基于实际数据重新评估阈值合理性：

1. 收集过去3个月实际数据
2. 计算各指标分布
3. 调整阈值区间
4. 更新风险标准

### 行业对标

参考行业平均水平动态调整：

```python
# 示例：如行业平均赔付率为90%
industry_avg = 90

# 我司标准可适当放宽
our_warning_threshold = industry_avg * 0.95  # 85.5%
our_danger_threshold = industry_avg * 1.05   # 94.5%
```
