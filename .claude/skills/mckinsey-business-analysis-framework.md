# 技能规范（最佳实践版）

## 使用场景
- 需要将业务问题结构化拆解为可量化的指标与分析路径。
- 与KPI、趋势、下钻、专题模块协同，形成闭环决策。

## 输入参数
- `problem`：问题陈述（中文）。
- `context`：业务背景与范围（字典）。
- `data_map`：指标与数据源映射（字典）。

## 输出
- 结构化分析计划：问题树、指标清单、数据需求与行动项。

## 流程
1. 问题树拆解（MECE）：目标-原因-证据-假设。
2. 数据映射：KPI/维度/时间/样本的对应关系。
3. 分析步骤：加载-计算-对比-结论-行动。

## 代码模板（含函数级中文注释）

```python
def build_mckinsey_plan(problem: str, context: dict, data_map: dict) -> dict:
    """
    函数用途：根据麦肯锡框架构建结构化分析计划
    参数说明：
    - problem: 中文问题陈述
    - context: 业务背景与范围字典
    - data_map: 指标与数据源映射字典
    返回结果：包含问题树、指标、数据需求与行动项的计划字典
    """
    plan = {
        'problem_tree': [f"目标: {problem}", "原因: 待验证", "证据: 待收集", "假设: 待检验"],
        'kpi_list': list(data_map.keys()),
        'data_requirements': data_map,
        'actions': ["加载与清洗数据", "计算并对比指标", "输出结论与建议"],
    }
    return plan
```
