# 核心KPI指标区间状态值配置

**文档版本**: 1.0  
**创建日期**: 2025-01-20  
**最后更新**: 2025-01-20  
**适用系统**: 车险数据分析平台 (Insuralytics)

---

## 📋 概述

本文档定义了车险数据分析平台中16个核心KPI指标的区间划分、状态等级和颜色编码规则。这些配置用于多维雷达图评分、趋势分析预警和KPI看板的颜色编码。

## 🎯 五级评分体系

所有指标均采用统一的**五级评分体系**，将业务指标转换为0-100标准化评分：

| 评分等级 | 评分范围 | 视觉反馈 | 颜色代码 | 业务含义 |
|---------|---------|---------|---------|----------|
| **卓越/优秀** | 95-100 | 深绿色 | #2E7D32 | 表现卓越，超出预期 |
| **健康/良好** | 86-94 | 浅绿色 | #4CAF50 | 表现良好，符合预期 |
| **中等** | 70-85 | 蓝色 | #1976D2 | 表现正常，可接受 |
| **预警/一般** | 40-69 | 黄色/橙色 | #FBC02D / #F57C00 | 需要关注，存在风险 |
| **危险/高危** | 0-39 | 红色 | #D32F2F | 严重风险，需立即处理 |

## 📊 核心比率指标区间设定

### 1. 满期边际贡献率 (contribution_margin_ratio)

**指标性质**: 正向指标（越高越好）  
**业务含义**: 反映业务盈利能力的核心指标  
**计算公式**: `100% - 变动成本率`

| 原始值范围 | 评分范围 | 等级 | 色彩 | 业务解读 |
|-----------|---------|------|------|----------|
| > 12% | 95-100 | 优秀 | 深绿 #2E7D32 | 盈利能力卓越 |
| 8-12% | 86-94 | 良好 | 浅绿 #4CAF50 | 盈利能力健康 |
| 6-8% | 70-85 | 中等 | 蓝色 #1976D2 | 盈利能力正常 |
| 4-6% | 40-69 | 一般 | 黄色 #FBC02D | 盈利空间有限 |
| 0-4% | 20-39 | 较差 | 橙色 #F57C00 | 盈利能力不足 |
| < 0% | 0-19 | 严重 | 红色 #D32F2F | 亏损状态 |

### 2. 保费时间进度达成率 (premium_time_progress_achievement_rate)

**指标性质**: 正向指标（越高越好）  
**业务含义**: 保费目标完成进度与时间进度的匹配度  
**计算公式**: `当期签单保费 / (年度目标保费 / 365 * 当期天数)`

| 原始值范围 | 评分范围 | 等级 | 色彩 | 业务解读 |
|-----------|---------|------|------|----------|
| ≥ 110% | 95-100 | 卓越 | 深绿 #2E7D32 | 超额完成目标 |
| 100-110% | 86-94 | 健康 | 浅绿 #4CAF50 | 按计划完成 |
| 90-100% | 70-85 | 预警 | 黄色 #FBC02D | 略低于预期 |
| 80-90% | 40-69 | 危险 | 橙色 #F57C00 | 明显滞后 |
| < 80% | 0-39 | 高危 | 红色 #D32F2F | 严重滞后 |

### 3. 满期赔付率 (loss_ratio)

**指标性质**: 逆向指标（越低越好）  
**业务含义**: 赔款支出占保费收入的比例  
**计算公式**: `满期赔付 / 满期保费`

| 原始值范围 | 评分范围 | 等级 | 色彩 | 业务解读 |
|-----------|---------|------|------|----------|
| < 50% | 95-100 | 优秀 | 深绿 #2E7D32 | 赔付控制优秀 |
| 50-60% | 86-94 | 良好 | 浅绿 #4CAF50 | 赔付控制良好 |
| 60-70% | 70-85 | 中等 | 蓝色 #1976D2 | 赔付控制正常 |
| 70-80% | 40-69 | 预警 | 黄色 #FBC02D | 赔付率偏高 |
| > 80% | 0-39 | 高危 | 红色 #D32F2F | 赔付率过高 |

### 4. 费用率 (expense_ratio)

**指标性质**: 逆向指标（越低越好）  
**业务含义**: 费用支出占保费收入的比例  
**计算公式**: `总费用 / 签单保费`

**计算公式**: `SUM(签单保费)`

### 7. 已报告赔款 (reported_claims_amount)

**指标性质**: 中性指标（无明确方向性）  
**业务含义**: 已发生并上报的赔案金额  
**计算公式**: `SUM(已报告赔款)`

*注：该指标主要用于监控，不设置明确的好坏区间*

### 8. 费用额 (expense_amount)

**指标性质**: 逆向指标（越低越好，但需结合业务规模）  
**业务含义**: 业务相关的总费用支出  
**计算公式**: `SUM(总费用)`

*注：该指标需结合签单保费规模综合评估*

## 🔧 技术实现

### 评分转换算法

系统采用**线性插值算法**实现评分平滑过渡：

```typescript
// 示例：满期边际贡献率评分转换
function convertContributionMarginRatio(value: number): number {
  if (value > 12) return 95 + (value - 12) * 5 / 8; // 95-100区间
  if (value >= 8) return 86 + (value - 8) * 9 / 4;  // 86-94区间
  if (value >= 6) return 70 + (value - 6) * 16 / 2; // 70-85区间
  if (value >= 4) return 40 + (value - 4) * 30 / 2; // 40-69区间
  if (value >= 0) return 20 + value * 20 / 4;       // 20-39区间
  return Math.max(0, 20 + value * 20 / 4);          // 0-19区间
}
```

### 综合评分计算

**多维雷达图综合评分** = 5个核心维度评分的算术平均值（四舍五入）

```typescript
const overallScore = Math.round(
  (contributionMarginScore + progressScore + lossRatioScore + 
   claimRatioScore + expenseRatioScore) / 5
);
```
